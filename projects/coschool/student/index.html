Coschool EdTech Dashboard Design UX Strategy

Student - Coschool

Transforming a static learning hub into an intelligent, adaptive assistant

Overview

Redesigned the core student dashboard for an EdTech platform serving 10,000+ students across 40 schools in India, shifting from a static, feature-heavy interface to an agent-driven, context-aware learning assistant. Collaborated with 4 Designers and cross-functional stakeholders.

Role

Lead UX Designer

Timeline

4 Months

01. The Problem Statement

As the platform scaled, multiple learning experiences were introduced—Homework, Self Study, Revision, Assessments, Edge cohort—leading to a fragmented and cluttered dashboard. A static dashboard failed to adapt to diverse student needs, resulting in poor discoverability, high cognitive load, and lack of clear next actions.

02. Goals & Success Metrics

Product Goals UX Goals Business Goals
Transition from tool → intelligent assistant Reduce cognitive overload Increase DAU and retention
Deliver context-aware experiences Guide students via a clear learning path Improve feature adoption
Improve feature discoverability Enable quick, actionable insights Reduce support queries

03. Research & Insights

Using session recordings (Microsoft Clarity), contextual observation, and feature usage analytics, we uncovered a key insight: Students don't want options — they want direction.

Discoverability Issues

Students frequently missed or ignored features due to low visibility.

Cognitive Overload

Too many options presented with equal priority caused decision paralysis.

Poor Navigation Model

Navigation relied on recall instead of recognition, making it unintuitive.

Lack of Direction

Students struggled to identify what to do next and what matters now.

04. Design Process & Transformation

We mapped student journeys across daily homework, exam prep, and concept learning. We then re-structured the IA around the academic timeline, learning intent, and user context over 5 design iterations.

Before: Static & Feature-Centric After: Contextual & Guided
Feature-heavy layout Context-aware sections (Homework, EDGE, Revision)
Equal visual weight for all actions Clear prioritization
No prioritization Action-oriented design
High cognitive load Reduced friction

05. The Solution

Agent-Driven Navigation

Replaced passive browsing with guided actions (e.g., 'Revise this chapter'). Reduced search effort and improved clarity.

Contextual Grouping

Content dynamically organized based on current academic phase and student progress. Ensures relevance and reduces clutter.

Progressive Disclosure

Only relevant features surfaced; secondary actions de-prioritized to eliminate overwhelm.

Actionable Insights Layer

Performance summaries, study patterns, and revision progress help students self-regulate learning.

Gamification & Ownership

Progress indicators and streak-based motivation encourage consistent engagement.

Breaking the Blank Canvas

Introduced recent activity, upcoming tasks, and smart suggestions to eliminate confusion at entry point.

06. Challenges & Trade-offs

Business vs UX

Balanced high feature visibility with simplicity via progressive disclosure and contextual surfacing.

Diverse User Needs

Designed for both high performers (exploration-driven) and struggling students (guidance-driven) using an adaptive, flexible layout.

Dynamic UI Complexity

Made the dashboard adaptive yet predictable through structured modular sections.

Paid vs Unpaid Experience

Showcased premium value without degrading the core experience, ensuring the homework flow remained uninterrupted.

07. Expected Impact & Reflection

Pre-launch expectations point to increased engagement through relevant content, better feature usage via visibility, decreased navigation time, and higher learning consistency.

Key Learning: Low-Resistance Paths

Users create their own 'low-resistance paths' — design should enable, not control. This led to thinking around flexible layouts and user-driven prioritization.

Key Learning: Designing at Scale

As features grow, UI must shift from feature-based to intent-based. Structure becomes more important than visuals.
